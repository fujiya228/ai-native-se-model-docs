## ビューの全体像

[なぜ複数ビューが必要か](/views/why-multiple-views/)で述べたように、仕事の流れを単一の図で表現しようとすると関心事が衝突する。本モデルでは、[基底モデルの要素](/foundation/base-model/)を共通の座標系として、以下のビューで投影する。

| # | ビュー名 | 焦点 | 主な問い |
|---|---|---|---|
| 1 | [ライフサイクルビュー](/views/view-1-lifecycle/) | 時間の流れ | どの順序で進むか？ |
| 2 | [実行設計ビュー](/views/view-2-execution/) | 責任と委譲 | 誰がどう進めるか？ |
| 3 | [アーティファクトビュー](/views/view-3-artifact/) | 受け渡し | 何を介して仕事が流れるか？ |
| 4 | [依存関係ビュー](/views/view-4-dependency/) | 待ちと競合 | 何が何に依存しているか？ |
| 5 | [制御環境ビュー](/views/view-5-control/) | 安全装置 | 何が安全性を担保しているか？ |
| 6 | [レイヤービュー](/views/view-6-layer/) | 表と裏 | 見える仕事と裏側の支援構造は何か？ |
| 7 | [測定ビュー](/views/view-7-measurement/) | 観測と改善 | 何をどう測るか？ |
| 8 | [As-Is / To-Be ビュー](/views/view-8-asis-tobe/) | 差分と検証 | 設計と実態はどれだけズレているか？ |

## ビューのメタ定義

各ビューは以下のメタ項目を持つ。これにより「何のために存在し、どこまで書けば十分か」が明確になる。

| メタ項目 | 説明 |
|---|---|
| **目的** | このビューは何を明らかにするためのものか。逆に何は答えないか |
| **主語** | 1行を何の単位で書くか（L1/L2ステップ、関係、指標 等） |
| **最小記述単位** | L1単位か、L2単位か、エッジ単位か |
| **記述項目** | そのビューで記述すべき列やフィールド |
| **停止基準** | どこまで書けば十分とみなすか |
| **ライフサイクルとの対応** | ライフサイクルのL1/L2とどう紐づくか |
| **いつ使うか** | どういう場面でこのビューを参照・更新するか |

<Aside type="tip">
メタ定義の中で特に重要なのは**停止基準**である。「どこまで書けば十分か」が不明確だと、ビューの記述が際限なく膨らむ。停止基準は「追加しても設計判断が変わらないなら十分」という考え方に基づく。
</Aside>

## ビューと基底モデル要素の対応

各ビューは、基底モデルの7要素のうち特定の側面にフォーカスする。

| ビュー | 主にフォーカスする基底要素 |
|---|---|
| ライフサイクル | Lifecycle Step |
| 実行設計 | Actor |
| アーティファクト | Artifact |
| 依存関係 | Dependency |
| 制御環境 | Control |
| レイヤー | Lifecycle Step + Control（レイヤー分割） |
| 測定 | Metric |
| As-Is / To-Be | 全要素（3面比較） |

## 推奨する最低セット

ビューすべてを一度に定義する必要はない。まず以下から始めることを推奨する。

### 最初に作る3ビュー

| 優先度 | ビュー | 理由 |
|---|---|---|
| 1 | ライフサイクルビュー | 全ビューの背骨。他のビューを載せる共通の下敷きになる |
| 2 | 実行設計ビュー | AI導入の核心。誰がどこまで自律するかを定義する |
| 3 | 依存関係ビュー | ボトルネック分析の基盤。AI導入による速度変化の影響を可視化する |

### 次に価値が高い2ビュー

| 優先度 | ビュー | 理由 |
|---|---|---|
| 4 | 測定ビュー | 設計が良いかどうかを実測で検証する。改善サイクルの基盤 |
| 5 | 制御環境ビュー | AI裁量レベルを上げる際の安全設計として不可欠 |

### 必要に応じて具体化するビュー

| ビュー | 具体化のタイミング |
|---|---|
| アーティファクトビュー | 受け渡しの品質問題が起きたとき、非同期協業の設計時 |
| レイヤービュー | 裏側の支援基盤（ハーネス、観測、評価系）を設計するとき |
| As-Is / To-Be ビュー | AI導入計画の策定時、導入後の効果検証時 |

<Aside>
優先度は一般的な推奨であり、コンテキストによって変わりうる。たとえば規制の強い業界では制御環境ビューの優先度が上がるし、大規模なチーム再編では実行設計ビューが最優先になる。
</Aside>
