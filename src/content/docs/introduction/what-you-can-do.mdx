## 使い方

本モデルには「記述的」「規範的」「変換的」の性質が同居しており、それぞれに対応する使い方（モード）がある。同じモデルを、目的に応じて異なるモードで使う。

| モード | 問い | 得られるもの |
|---|---|---|
| **記述モード** | 今の仕事はどう流れているか？ | 現状の構造化された理解 |
| **設計モード** | 仕事をどう設計すべきか？ | 設計意図の明文化 |
| **変換モード** | AI導入で何がどう変わるか？ | AI導入の具体的な設計判断 |

---

## 記述モード: 現状を分析する

モデルの語彙（ライフサイクル、基底要素、設計属性、ビュー）を使って、現在のワークフローを構造的に記述する。

<Card title="シナリオ: 開発プロセスの可視化">
あるチームが「うちの開発プロセスは属人化していて説明できない」と感じている。記述モードでは、以下のようにモデルの語彙を使って現状を構造化する。

1. 既存の開発プロセスを**ライフサイクルビュー**にマッピングし、どの段階に対応するかを明確にする
2. **実行設計ビュー**で現状のRACIを可視化し、「誰が何の責任を持っているか」を明示する
3. **依存関係ビュー**でボトルネックを特定し、「どこで待ちが生じているか」を見る
4. **測定ビュー**で現在のリードタイムやスループットを記録する

結果として得られるのは、**「何が見えていなかったか」の発見**である。「実はコードレビューで3日待ちが発生していた」「デプロイ承認が1人に集中していた」といった構造的な問題が見えてくる。
</Card>

## 設計モード: あるべき姿を定義する

設計属性を各ステップに対して定義し、ビューを埋めることで、目標とするワークフローを設計する。

<Card title="シナリオ: 新プロジェクトの開発プロセス設計">
新規プロジェクトの立ち上げで、開発プロセスを最初からきちんと設計したい。設計モードでは以下を行う。

1. 各L1段階（Requirements, Design, Implementation, ...）に対して**実行設計**を定義する — 誰がResponsibleで誰がAccountableか
2. **制御環境ビュー**で安全設計を行う — CI/CDパイプラインのどこにゲートを置くか、自動テストの基準は何か
3. **測定ビュー**で目標指標を設定する — PRのレビュー待ち時間の上限、デプロイ頻度の目標値

結果として得られるのは、**チーム間で共有可能な設計図**である。新メンバーが「うちのチームではどう開発するのか」を構造的に理解できる。
</Card>

## 変換モード: AI導入前後の差分を設計する

As-Is/To-Beビューを中心に、AI導入前後の差分を設計する。裁量レベルの引き上げ、実行主体の変更、制御環境の再設計がこのモードの主な活動になる。

<Card title="シナリオ: コードレビューのAIエージェント導入">
「コードレビューにAIエージェントを導入したい」というケースを考える。変換モードでは以下の手順で設計する。

1. **As-Is（現状）** を記述モードで整理する — 現在は人間のレビュアーが全PRをレビュー、平均待ち時間2日、レビュアーが3人に集中
2. **To-Be（目標）** を設計モードで定義する — AIエージェントが初回レビュー（スタイル、セキュリティ、テストカバレッジ）を実行、人間はロジックと設計判断に集中、裁量レベルL2（計画に沿って自律実行、結果を報告）
3. **差分**を特定する — 実行主体が「Human only」から「AI Agent + Human」に変わる、制御環境に自動レビューの品質基準を追加する必要がある
4. **Validation** で検証する — 導入後のレビュー待ち時間、指摘の見落とし率を実測データと突き合わせる

結果として得られるのは、**「何を変え、何を変えないか」の明示的な設計判断**である。「AIに任せる範囲」「人が見るべき範囲」「安全装置として何を置くか」が構造的に定義される。
</Card>

## モード間の関係

モードは排他的ではない。実際の運用では、典型的に以下のサイクルで使われる。

```
記述モード（現状把握）
  → 変換モード（差分設計）
    → 設計モード（目標定義）
      → 記述モード（実測・検証）
```

<Aside type="tip">
View 8（As-Is / To-Be / Validation）は、このサイクルを1つのビューの中で回すための仕組みである。詳しくは[View 8: As-Is / To-Be / Validation ビュー](/views/view-8-asis-tobe/)を参照する。
</Aside>
