## 目的

各ステップを**誰がどう進めるか**を定義する。責任分担、委譲範囲、承認の在り方を明らかにする。

このビューは「依存関係」や「測定指標」には答えない。それらは依存関係ビューや測定ビューの役割である。

## メタ定義

| メタ項目 | 定義 |
|---|---|
| **目的** | 各ステップを誰がどう進めるかを定義する。責任分担、委譲範囲、承認の在り方を明らかにする |
| **主語** | L1 / L2 ステップ |
| **最小記述単位** | L2（高優先L1）または L1（粗め扱いL1） |
| **記述項目** | RACI（R / A / C / I）、裁量レベル（L0〜L4）、トリガー、ゲート、フォールバック、入力アーティファクト、出力アーティファクト、観測点 |
| **停止基準** | 追加しても**役割分担や委譲判断が増えない**なら十分 |
| **ライフサイクルとの対応** | L1/L2ステップごとに1行。ライフサイクルビューの各ステップに RACI 等を重ねる形 |
| **いつ使うか** | AI導入の役割設計時、チーム体制変更時、承認フロー見直し時 |

## 表現するもの

### 実行主体と責任主体

[実行主体と責任主体](/execution/actor-and-responsibility/)で定義した4種類の Actor（Human / AI Agent / Automation / External）を、各ステップに割り当てる。

### RACI と裁量レベル

[RACIと裁量レベル](/execution/raci-and-discretion/)で定義した道具を使い、各ステップの責任分担と委譲の程度を記述する。

### 記述項目の全体像

実行設計ビューで各ステップに記述する項目は[実行設計ビューの推奨項目](/execution/raci-and-discretion/#実行設計ビューの推奨項目)で定義した8項目である。

| # | 項目 | このビューでの役割 |
|---|---|---|
| 1 | RACI | 実行・責任・相談・共有の割り当て |
| 2 | 裁量レベル | L0〜L4 の設定 |
| 3 | トリガー | このステップの開始条件 |
| 4 | ゲート | 次のステップに進むための条件 |
| 5 | フォールバック | AIが失敗したときの代替手順 |
| 6 | 入力アーティファクト | このステップが受け取るもの |
| 7 | 出力アーティファクト | このステップが生成するもの |
| 8 | 観測点 | このステップで計測すべきもの |

<Aside type="tip">
すべてのステップで8項目すべてを埋める必要はない。AIの裁量を広げるステップほど、ゲート・フォールバック・観測点の設計を意識的に行う。
</Aside>

## AIネイティブでの変化

実行設計ビューは、AI導入によって最も大きく変化するビューである。

| 変化 | 説明 |
|---|---|
| **R に AI Agent が入る** | 従来 Human だけだった実行を AI が担う |
| **裁量レベルが上がる** | L0（提案のみ）から L2〜L3（自律実行）へ移行する |
| **I の重要性が上がる** | AIの実行結果を「誰に、どう共有するか」の設計が重要になる |
| **逐次承認が減り、事後確認が増える** | 人の関与が事前承認から事後監視へシフトする |

この変化は、[制御環境ビュー](/views/view-5-control/)と[測定ビュー](/views/view-7-measurement/)の設計なしには安全に進められない。
